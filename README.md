# 🎯 AI 워크샵 스태프 플랫폼

지침서 기반으로 구현한 Node.js + Express 웹 앱입니다. 관리자/참여자 화면이 분리되어 있고,
모든 데이터는 `data/db.json` 파일에 저장됩니다(별도 DB 불필요).

## 실행 방법

```bash
cd ai-workshop-staff
npm install
npm start
```

- 참여자(조장) 화면: `http://localhost:3000/` (또는 `index.html`)
- 관리자 화면: `http://localhost:3000/admin.html`
- 기본 관리자 비밀번호: `admin1234` (관리자 화면 우측 상단 → 로그인 후 `PUT /api/admin/password`로 변경 가능. 추후 설정 UI 추가 예정 전까지는 `data/db.json`의 `config.adminPassword` 값을 직접 수정해도 됩니다)

워크샵 당일 여러 조가 동시에 접속하려면, 관리자 PC를 같은 네트워크의 서버로 띄우고
참여자들에게 `http://<관리자PC의 IP>:3000` 주소를 공유하면 됩니다.

## 사용 순서 (관리자)

1. **참여자 / 조 관리** 탭에서 조(팀)를 추가하고, 참여자(이름+부여코드+소속조+조장여부)를 등록
2. **미션 풀** 탭에서 랜덤 미션(2단계 이후 사용)과 점심식사 미션(반드시 2번째 미션에 배정)을 등록
3. **힌트 풀 / 자동발송** 탭에서 힌트를 등록하고, 필요 시 "N회 성공 시 특정 힌트 자동 발송" 조건 추가
4. 워크샵 시작 → 참여자들이 `index.html`에서 이름+부여코드 입력 → 본인 조 확인
   - 확인 후 전원에게 "사진을 찍어 조장을 확인해보세요!" 안내와 함께 같은 조원들과의 인증사진 업로드 화면이 표시됨
   - **진행 모니터링** 탭에서 해당 조의 "0단계 · 조원 찾기 인증사진"을 확인 후 [✅ 조원찾기 승인] 처리
   - 승인되면: 조장은 다음 단계(랜덤 미션)로 진입, 조장이 아닌 참여자는 "다음 미션부터는 조장만 진행할 수 있습니다. 조장의 기기를 확인해주세요 📲" 안내가 표시되고 더 진행할 수 없음
5. **진행 모니터링** 탭에서 각 조의 제출물(사진/영상/링크)을 확인하고 [승인] 또는 [반려] 처리
   - 승인 시: 성공 횟수 +1, 힌트 풀에서 랜덤 힌트 1개 자동 지급, 자동발송 조건 충족 시 추가 힌트 지급

## 참여자 플로우

1. 이름 + 부여코드 입력 → 소속 조 확인 (보안 안내 문구 표시)
2. **0단계 (전원)**: "사진을 찍어 조장을 확인해보세요!" → 같은 조원들과 함께 찍은 사진/영상 업로드 또는 링크 제출 → "관리자 확인 중..." 대기
3. 관리자 승인 후
   - **조장**: 다음 단계(랜덤 미션)로 자동 진입
   - **조장이 아닌 참여자**: "다음 미션부터는 조장만 진행할 수 있습니다. 조장의 기기를 확인해주세요 📲" 안내 표시, 더 진행 불가

## 조장 플로우 (2단계 이후)

1. [미션 받기] → 랜덤 미션 카드 표시 (2번째 미션은 항상 점심식사 미션 풀에서 배정)
2. [미션 재추첨] — 워크샵 전체에서 **1회만** 가능 (사용 후 버튼 비활성화)
3. 사진/영상 업로드 또는 외부 링크 입력 후 제출 → "관리자 확인 중..." 대기
4. 관리자 승인 시 자동으로 힌트 카드 표시 (랜덤 힌트 + 조건 충족 시 자동발송 힌트)

## 데이터 구조

- `data/db.json` : 참여자, 조, 미션 풀, 힌트 풀, 조별 진행상태, 힌트 발송 로그를 모두 보관
- `data/uploads/` : 참여자가 업로드한 사진/영상 파일 저장 위치

## 보안/주의사항

- 관리자 비밀번호는 평문으로 `db.json`에 저장되므로, 외부에 공개되는 서버에 올릴 경우 반드시 변경하세요.
- 조별 미션·힌트 정보는 해당 조의 화면에만 노출되며, 다른 조 화면에서는 조회할 수 없습니다.
